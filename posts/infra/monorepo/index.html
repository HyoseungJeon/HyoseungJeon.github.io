    💡
    롯데카드 프로젝트에 투입되며 환경을 분석하던 중 MonoRepo에 대해 듣게되었다. 소스를 받아보니 모듈별 소스가 한 프로젝트로 관리되고 있었다! 신기하네..

    정의#

    독립된 여러 프로젝트나 라이브러리를 하나의 저장소에서 관리하는 방식을 의미합니다.

    정의는 굉장히 간단하다. 그냥 대충 프로젝트 단위의 소스를 하나의 디렉토리로 관리하고 그 디렉토리를 기준으로 Git을 올린다고 생각하면 편할 듯 하다.

    장단점#

    장점#

    1. 코드 공유·재사용 용이 – 공통 라이브러리나 유틸을 하나의 저장소에서 바로 참조 가능.
    2. 일관된 버전 관리 – 모든 프로젝트가 동일한 버전 규칙과 릴리스 사이클을 사용.
    3. 일괄 변경 가능 – 여러 서비스·패키지에 걸친 대규모 변경을 한 커밋으로 처리 가능.
    4. 중앙 집중형 빌드/테스트 – CI/CD 파이프라인을 통합 관리 가능.
    5. 일관된 개발 환경 – 같은 린터, 포맷터, 빌드 도구 설정을 강제 가능.

    단점#

    1. 저장소 규모 커짐 – 히스토리·파일 수 증가로 Git 성능 저하 가능.
    2. 빌드/테스트 비용 증가 – 변경 영향 범위가 커서 불필요한 빌드·테스트가 발생할 수 있음.
    3. 권한 관리 어려움 – 프로젝트별 접근 제한이 까다로움.
    4. 변경 충돌 가능성 증가 – 많은 팀이 같은 저장소에서 작업하므로 머지 충돌 빈도가 높아짐.
    5. 도구·인프라 의존성 – 대규모 Monorepo를 효율적으로 운영하려면 Bazel, Nx 같은 빌드 도구와 강력한 CI 환경 필요

    내가 봤을 땐 일괄 변경과 일관된 개발 환경 등이 가장 큰 장점으로 보인다.

    현재 모듈별로 프로젝트를 관리 시에는 lib 의존성이 시간이 지날수록 서버별로 다 달라져서 서서히 문제가 발생하던 것 같다.

    또한 commit시에 수정한 프로젝트별로 브랜치 생성 및 버전 관리까지 가져가야 하기 때문에 꽤나 휴먼에러에 쉽게 노출되는 부분인 것 같다.

    단점으로는 권한 관리와 저장소 규모, 도구 인프라 의존성이 큰 것 같다.

    하나의 디렉토리로 관리하다보니 소스를 받으면 실제로 내가 담당하는 영역보다 불필요하게 많은 양을 받게 된다. 물론 IDE의 ignore 설정으로 어느정도 성능적인 영향도는 줄일 수 있던 것 같다.

    중간 관리자가 프로젝트를 관리하는 데 권한으로 제어를 한다 해도 어쩔 수 없이 통합된 환경 때문에 발생하는 휴먼에러를 놓치기 쉬운 것 같다. 하지만 이 또한 파이프라인에서 각 브랜치별 권한에 따라의 소스 수정 범위를 체크하는 프로세스를 넣는다면 괜찮을 것 같기도..?

    가장 큰 문제는 인프라 의존성이다. 사실 monorepo의 가장 문제점으로 보인다. 물론 인프라가 기능을 잘 지원하기 때문에 동작 문제보다는 지나치게 의존성 관계가 커진다는 점이다. pnpm처럼 package 기능이 없다면 불가능하지 않나..? 싶다

    내 생각#

    현재 프로젝트에서는 front-end 프로젝트만을 묶어서 monorepo 형태로 관리하고 있었다. 아마도 통합앱이다 보니 더더욱 일관된 코드를 지향하기 위한 이유로 보였다. 헌데 경험상으로 느낀 문제점은, 사전에 공통 함수, 공통 css 등을 사용하는 환경에서는 모든 서버들이 다 영향도가 있다. 헌데, 만약 해당 코드의 수정이 사전 공지가 없다면 각 프로젝트에서는 잘 돌아가던 서버에 갑자기 이슈가 리포팅 될 때가 있다. n-1개의 프로젝트에서 문제가 없더라도, 1개의 프로젝트에서 문제가 될 수 있는 일이 많아서 간단한 코드 수정도 그 영향도가 큰 경우가 많은 것 같더라. 하여 전체 영향도를 갖고 있는 작업자는 그만큼 해당 애로사항을 고려하여 작업해야 할 것 같다. 그리고 git에서도 rebase 등의 작업을 주기적으로 하여 형상관리가 힘들어 보이더라. 사실 장점보단 단점이 더 눈에 띄었던 것 같다. 실제로 공통적으로 관리하는 단위가 함수다 보니, 실효는 없다고 느껴졌다. 정말 만약 composible, component 단위로 했다면 얘기가 달랐겠지만, 딱히 잘..?
