    정의#

    CDN이란?#

    콘텐츠 전송 네트워크(Content Delivery Network)는 데이터 사용량이 많은 애플리케이션의 웹 페이지 로드 속도를 높이는 상호 연결된 서버 네트워크입니다. CDN은 콘텐츠 전송 네트워크 또는 콘텐츠 배포 네트워크를 의미할 수 있습니다. 사용자가 웹 사이트를 방문할 때 해당 웹 사이트 서버의 데이터는 사용자의 컴퓨터에 도달하기 위해 인터넷을 통해 이동해야 합니다. 사용자가 해당 서버에서 멀리 떨어져 있는 경우 동영상 또는 웹 사이트 이미지와 같은 대용량 파일을 로드하는 데 시간이 오래 걸립니다. 대신 웹 사이트 콘텐츠는 지리적으로 사용자와 가까운 CDN 서버에 저장되며 컴퓨터에 훨씬 빨리 도달합니다. ref by https://aws.amazon.com/ko/what-is/cdn/

    정리하면 중간 Caching Server를 두어 실제 물리적 거리를 줄여 네트워크 응답 속도를 개선하고 중간 서버를 통해 비용 절감, 보안 등의 추가적인 장점을 누릴 수 있는 Network 환경을 말한다!

    장점#

    • 페이지 로드 시간 단축
    • 대역폭 비용 절감
    • 콘텐츠 가용성 제고
    • 웹 사이트 보안 강화

    주요 프로세스#

    • 캐싱
    • 동적 가속 : 동적 데이터 로딩은 캐싱의 동작이 불분명하다. 하여 모든 동적 요청에 대해 오리진 서버와 연결을 최적화 하여 동적 데이터 응답 시간을 단축한다.
    • 엣지 로직 계산 : 클라이언트 - 서버 통신을 최적화 한다.
      • 사용자 요청 검사, 캐싱 동작 수정
      • 잘못된 사용자 요청 처리
      • 응답 전 콘텐츠 최적화

    내 생각#

    본 프로젝트는 프론트 서버를 CDN 환경으로? 배포하여 화면 응답 속도를 개선하려 했다! 근데 내 생각엔.. 실질적으로 본 서비스는 국내 사용자에게만 제공되는 화면이고, 데이터의 양 자체도 화면 소스가 전부라 유의미한 응답속도 개선이 기대되지 않는 것 같은데.. 실제 CDN 적용으로 유의미한 개선 데이터가 쌓인지는 보지 못했다 ㅠㅜ.. 기회가 된다면 확인하고 싶다!!

    Nuxt3로 랜더링 모드를 SSG로 하는 것에 대해 의문이 많았는데..! 실제로 위 Network를 활용을 위해 정적 Content로 배포가 필요하여 해당 모드를 선택했는가..? 에 대한 의문점도 남아있다! 실제로 Nuxt로 랜더링 시, HTML File의 무게가 상당하긴 했던 것 같다!

    비단 현재 프로젝트 뿐만 아니라, Web Network 환경에서 필수적인 지식 같다!! 이번 기회로 알게 되어서 너무 기쁘다.. 어디가서 모를뻔 😂
