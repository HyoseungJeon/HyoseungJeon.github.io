    LDAP이란?#

    경량 디렉터리 액세스 프로토콜(영어: Lightweight Directory Access Protocol; LDAP)은 TCP/IP 위에서 디렉터리 서비스를 조회하고 수정하는 응용 프로토콜이다.

    디렉터리 서비스?

    디렉터리 서비스는 네트워크상에서 조직, 사용자, 파일, 디바이스 등의 정보를 저장하고 관리하는 시스템입니다. 전화번호부와 비슷한 개념으로 생각할 수 있습니다.

    주요 특징:

    주요 특징#

    • 계층적 구조로 정보를 저장하고 관리
    • 빠른 읽기 작업에 최적화
    • 중앙집중식 정보 관리 제공
    • 네트워크 자원에 대한 접근 제어 가능

    결론적으론 계층화된 데이터를 관리하는 하나의 서비스라고 보면 이해가 빠른 것 같다. 또한, 표준 프로토콜이기 때문에 다양한 소프트웨어와 운영 체제에 이식이 가능하다는 장점이 있다.

    권한 관리 기능 또한 자동으로 지원되어 따로 구현하거나 다른 서비스를 붙일 필요가 없다.

    한계점#

    • 복잡한 데이터 모델 적용 불가
    • 기능 확장성이 없다
    • 트랜잭션 기능이 없다.
    • 엄격한 스키마 룰

    위와 같은 특징들로 인해 LDAP은 주로 조직 관리와 같은 특정 용도에 최적화 되어있다.

    내 생각#

    회원 모듈의 인수인계를 받으면서 처음 들어보는 단어였다! 😒 LDAP…? 엘 다압..? 대충 느낌으로 조직 데이터를 저 서비스를 통해서 조회해오는 구나 라는 맥락까지만 이해하고 넘어갔는데 궁금해서 찾아보았다! 근데 생각보다 도움 되는 정보였다.. 보통 대규모의 조직은 LDAP 시스템을 통해 조직 데이터를 관리하는 듯 했다! 특히 계열사를 가진..? 대기업의 경우 조직 데이터를 관리하는 것이 애로사항이 많아서 그런지 명확한 목적성이 있는 서비스를 활용하는 것이 보안, 관리 측면에서의 전체 비용이 매우 좋아보였다. 특히나 이식율이 높다는게 가장 인상깊었다! 프로토콜 개념이라 다양한 환경에 이식이 가능한 점이 사실 해당 프로토콜을 활용하는 가장 큰 이유가 아닐까 싶었다.
