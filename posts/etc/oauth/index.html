    OAuth#

    정의#

    OAuth 2.0 is the industry-standard protocol for authorization. OAuth 2.0 focuses on client developer simplicity while providing specific authorization flows for web applications, desktop applications, mobile phones, and living room devices. This specification and its extensions are being developed within the IETF OAuth Working Group. 출처 - OAuth 2.0 — OAuth

    OAuth은 특정한 서비스를 말하는 것이 아닌 하나의 표준으로 사용자의 인증 환경에 대한 흐름을 정의하는 역할이다!

    사용자로 하여금 외부 인증 사이트를 통하여 간편하게 로그인 하거나, 회원가입 할 수 있다는 장점이 있다! 개발자 또한 표준을 통하여 간편하게 외부 연동 인증 기능을 구현할 수 있다!

    구성 요소#

    • Resource Owner : 사용자
    • Client : 서비스 제공 서버
    • Authorization Server : 인증 서버
    • Resource Server : 인증사의 정보 서버
    • Access Token : 인증 서버에서 발급한 token
    • Refresh Token : Access Token 만료 시 token 갱신을 위한 token, 만료 기간이 상대적으로 길다.

    내 생각#

    요즘 웹사이트를 이용하다 보면 대부분이 네이버, 카카오톡으로 로그인 하기 등의 기능을 지원하고 있다! 하여 해당 기능이 구체적으로 어떤 방식으로 구현되었는지 알아보게 되어 재미있었다! 사실 기술적으로 특별한 면이 보이진 않았지만 사용자의 데이터를 이미 소지하고 있는 서비스를 통해 사용자는 손쉽게 회원 가입, 로그인 등을 할 수 있다는 점이 매우 매력포인트로 보였다. 또한 보안 관점에서도 이는 큰 메리트로 보였다!! 네트워크 패킷 해킹 외에 인증 과정 자체를 다른 서버에 위임할 수 있다는 장점도 있지 않나..? 내 입장에서는 token만 잘 관리하면 나머지 문제에서는 조금 벗어날 수 있는 것 처럼 보였다! 본 프로젝트는 회사의 규모가 컸다. 헌데 계정을 관리하는 하나의 서비스를 통해 전체 계열사의 계정 인증을 처리할 수 있었다! 따로 사용료를 받지 않는 이상.. 아주 좋은 기술? 프로토콜? 페러다임? 인 것 같다!
