    Nuxt 3에서 composables 디렉토리는 Vue 3의 Composition API를 사용하여 재사용 가능한 함수를 정의하고 관리하는 역할을 합니다. 이 디렉토리는 컴포넌트 간에 공통 로직을 공유하고 재사용할 수 있는 방법을 제공합니다. **composables**는 Vue 3의 Composition API와 밀접한 관련이 있으며, Nuxt 3 프로젝트에서 코드 구조를 더욱 깔끔하고 유지보수하기 쉽게 만듭니다.

    주요 역할#

    1. 재사용 가능한 로직 관리: composables 디렉토리 내에 정의된 함수들은 다양한 컴포넌트에서 재사용할 수 있는 로직을 캡슐화합니다. 예를 들어, API 호출, 상태 관리, 데이터 처리 등을 위한 공통 로직을 composable 함수로 작성할 수 있습니다.
    2. 상태 관리: composables 디렉토리에서는 컴포넌트 간에 공유되는 상태를 관리하는데 유용합니다. Vue의 Composition API를 활용하여 반응형 상태를 관리하고, 이를 여러 컴포넌트에서 사용할 수 있게 합니다.
    3. 코드 구조화: composables 디렉토리를 사용하면 프로젝트의 코드 구조를 더 모듈화하고, 관련 로직을 한 곳에 모아서 관리할 수 있습니다. 이는 코드의 가독성과 유지보수성을 크게 향상시킵니다.

    예시: composables 디렉토리 사용#

    1. 디렉토리 구조: Nuxt 3 프로젝트에서 composables 디렉토리는 프로젝트 루트에 위치합니다.

      arduino코드 복사
my-nuxt3-app/
├── composables/
│   ├── useCounter.js
│   └── useFetchData.js
├── pages/
├── components/
└── nuxt.config.js

    2. Composable 함수 정의: 예를 들어, 카운터를 관리하는 **useCounter**라는 composable 함수를 정의할 수 있습니다.

      javascript코드 복사
// composables/useCounter.js
import { ref } from 'vue';

export function useCounter() {
  const count = ref(0);

  function increment() {
    count.value++;
  }

  function decrement() {
    count.value--;
  }

  return {
    count,
    increment,
    decrement
  };
}

    3. Composable 함수 사용: 정의한 composable 함수를 Nuxt 3 컴포넌트에서 사용할 수 있습니다.

      vue코드 복사
<!-- pages/index.vue -->
<template>
  <div>
    <p>Count: {{ count }}</p>
    <button @click="increment">Increment</button>
    <button @click="decrement">Decrement</button>
  </div>
</template>

<script setup>
import { useCounter } from '~/composables/useCounter';

const { count, increment, decrement } = useCounter();
</script>

    4. API 데이터 페칭: API 데이터를 페칭하는 **useFetchData**라는 composable 함수를 정의할 수 있습니다.

      javascript코드 복사
// composables/useFetchData.js
import { ref } from 'vue';
import axios from 'axios';

export function useFetchData(url) {
  const data = ref(null);
  const error = ref(null);
  const loading = ref(false);

  async function fetchData() {
    loading.value = true;
    try {
      const response = await axios.get(url);
      data.value = response.data;
    } catch (err) {
      error.value = err;
    } finally {
      loading.value = false;
    }
  }

  fetchData();

  return {
    data,
    error,
    loading,
    refetch: fetchData
  };
}

    5. Composable 함수 사용: 데이터 페칭 로직을 컴포넌트에서 사용할 수 있습니다.

      vue코드 복사
<!-- pages/data.vue -->
<template>
  <div>
    <div v-if="loading">Loading...</div>
    <div v-else-if="error">{{ error.message }}</div>
    <div v-else>{{ data }}</div>
    <button @click="refetch">Refetch</button>
  </div>
</template>

<script setup>
import { useFetchData } from '~/composables/useFetchData';

const { data, error, loading, refetch } = useFetchData('https://api.example.com/data');
</script>

    요약#

    • composables 디렉토리는 Nuxt 3에서 재사용 가능한 함수를 정의하는 곳입니다.
    • Composable 함수는 Vue 3의 Composition API를 사용하여 작성되며, 컴포넌트 간에 공유될 수 있는 로직을 캡슐화합니다.
    • 주요 역할에는 재사용 가능한 로직 관리, 상태 관리, 코드 구조화 등이 포함됩니다.
    • 사용 예시로는 상태 관리, API 데이터 페칭 등이 있으며, 이를 통해 프로젝트의 모듈성과 유지보수성을 향상시킬 수 있습니다.

    composables 디렉토리를 활용하면 Nuxt 3 프로젝트에서 코드의 일관성과 재사용성을 높일 수 있습니다.
