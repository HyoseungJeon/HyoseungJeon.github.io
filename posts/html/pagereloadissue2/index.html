    Back Office 페이지에서 또 똑같은 issue가 발생했다! 내용은 grid 페이지에서 검색어 란에 focus 상태에서 enter 키 입력 시 page가 reload 되는데 살펴보자!

    문제 현상#

    문제는 간단하다! 검색어를 입력 후 enter 키를 누를 시 페이지가 reload 되고 있다..

    원인#

    문제의 시발점은 최근 엑셀 다운로드 기능 개선이였다! 기존에 본 이슈의 해결을 위해 초기화 부분에서 jQuery를 이용해 form event를 변조하고 있었는데, 업데이트 된 다운로드 방식은 해당 form element의 input 값을 이용하기 위하여 해당 form을 그대로 재사용 하는 방식이였다! 하여 submit 기능이 필요하였는데 jquery를 통해 선언된 소스 때문에 submit 기능을 사용할 수 없어 초기화 부분의 소스를 지웠다! (이 당시에는 위와 같은 side effect를 몰랐다.. 😥) 그 후에는 form Element 안의 <input type="text"> element에 focus 상태에서 enter 키 입력 시 자동으로 submit event가 수행되어 page가 reload 되고 있었던 것이다! (이 내용도.. 😋)

    해결 방법#

    해결법은 간단했다! 해당하는 form element에 선언적으로 onsubmit event를 할당하는 것으로 끝이였다!

    느낀점#

    사소한 사항인데.. 확실히 놓치고 있었던 내용이라 발생했던 이슈인 것 같았다.. 초기화에 있던 소스를 지울 때 왜 이런 소스가 있지..? 라는 생각만 하고 따로 side effect가 보이지 않아 지웠는데.. 이러한 이슈가 있었다니 당황스러웠다!! 확실히.. 이유 없는 소스는 없는 것 같다! 비록 히스토리가 지나며 정말 필요없는 소스들도 있긴 하지만 이 처럼 필요 없어 보이지만 중요한 역할을 하는 소스도 있다!! (간단하게 주석이라도 있었으면 좋지 않았을까.. 🤐) 하여 만약 위와 같이 소스 안에서 그 히스토리를 추격하기 어렵거나, 코드만으로 납득되지 않는 소스가 있을 때엔 주석도 함께 달아주는 습관을 기르면 좋겠다는 생각이 들었다!! n차 피해자 방지.. 😎
